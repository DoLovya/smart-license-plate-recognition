# 智能车牌检测识别系统-项目规划

## 1. 项目概述

- 项目名称：`智能车牌检测识别系统`
- 项目定位：面向桌面端部署的车牌检测识别工具，突出 `Qt/C++ 客户端`、`模型推理部署`、`图像可视化交互` 与 `工程化交付`
- 项目目标：补强当前简历中“AI 视觉应用落地”和“模型部署工程化”信号，同时保持与既有优势 `工业视觉客户端开发 / 平台建设 / 性能优化` 一致
- 输出形式：
  - 一套可运行的桌面端应用
  - 一个可公开展示的 GitHub 项目
  - 一段可直接写入简历的项目经历

## 2. 项目价值

### 对求职竞争力的价值

- 强化 `C++/Qt + 视觉算法落地 + 推理部署` 的组合能力
- 证明具备从 `模型接入 -> 客户端展示 -> 结果导出 -> 工程化交付` 的完整闭环能力
- 为 `工业视觉客户端开发`、`AI 应用工程`、`推理部署/视觉工具开发` 等岗位提供更直接的项目证据

### 对当前简历的补强点

- 当前简历优势集中在 `客户端架构`、`平台建设`、`性能优化`
- 当前简历相对薄弱的是 `可独立展示的视觉 AI 应用项目`
- 本项目的核心作用不是新增一个“算法 demo”，而是补足一个“工程化视觉系统”

## 3. 项目目标

### 核心目标

- 实现车牌检测与字符识别的一体化流程
- 支持图片、文件夹批量处理，输出结构化识别结果
- 提供 Qt 客户端界面，实现图像展示、检测框绘制、结果列表与导出
- 完成模型推理接入与基础性能优化

### 扩展目标

- 支持视频流或摄像头实时识别
- 支持多线程异步推理，优化 UI 卡顿问题
- 支持模型切换、阈值调整、日志输出
- 提供 CPU/GPU 推理性能对比数据

## 4. 项目范围

### 本期范围

- 车牌检测
- 车牌字符识别
- 图片加载与结果展示
- 批量处理
- 识别结果导出
- 推理日志与错误提示
- 基础性能统计

### 暂不纳入

- 端到端训练框架完整重构
- 云端服务、用户系统、多端同步
- 复杂权限系统
- 商业化部署

## 5. 技术路线

- 客户端：`C++`、`Qt`

- 构建系统：`CMake`
- 数据存储：`SQLite` 或 `CSV/JSON`
- 工程辅助：`Git`、`CI/CD`

  - 结果后处理
- `基础设施层`
  - 配置管理

## 7. 工程亮点设计

### 需要刻意做出来的亮点

- `客户端工程化`
  - 使用 Qt 构建完整桌面交互，而不是仅命令行推理
- `推理部署闭环`
  - 从 PyTorch 到 ONNX Runtime 的完整推理接入
- `性能优化`
  - 采用异步任务或生产者-消费者模型避免 UI 阻塞
- `可视化能力`
  - 检测框叠加、识别结果面板、批量处理结果展示
- `交付能力`
  - 提供 Release 构建、配置说明与运行截图

### 面试时可重点讲的技术点

- Qt 图像显示与绘制叠加层的实现方式
- 模型前处理/后处理流程设计
- ONNX Runtime 推理流程封装
- 多线程任务队列与 UI 解耦
- 批量处理时的性能瓶颈定位与优化

## 10. 风险与应对

### 风险 1：模型效果不稳定

- 应对：优先选择成熟公开模型或可复现方案，先完成工程闭环，再逐步优化模型效果

### 风险 2：客户端接入复杂度高

- 应对：先做命令行推理验证，再逐步迁移到 Qt 界面

### 风险 3：项目过于偏算法，削弱个人主线

- 应对：始终围绕 `客户端 + 部署 + 可视化 + 性能` 展开，不把重点放在纯训练调参

### 风险 4：周期过长，迟迟无法形成简历成果

- 应对：采用分阶段交付，优先做出可展示的 MVP
